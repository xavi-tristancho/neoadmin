# Quick Start

## Initial Set Up

:::tip

This library is optimized to be used through <WrapperLink url="https://reactjs.org/docs/create-a-new-react-app.html#create-react-app">`npx create-react-app my-app` </WrapperLink >

:::

Foremost you can delete the `App.js` file, then you can access the `index.js` and import the `App` component provided by the library and some <LocalLink url="#" text="contexts"/> <!-- /api/contexts --> to wrap the `<App />`. The result should be something like this:

```jsx title="/src/index.js"
import React from "react";
import ReactDOM from "react-dom";
import reportWebVitals from "./reportWebVitals";
// highlight-next-line
import { apps, contexts } from "@app-artisans/backoffice";

// highlight-start

const { AuthProvider, ConfigProvider } = contexts;
// highlight-end

ReactDOM.render(
  <React.StrictMode>
    // highlight-start
    <AuthProvider>
      <ConfigProvider>
        <App />
      </ConfigProvider>
    </AuthProvider>
    // highlight-end
  </React.StrictMode>,
  document.getElementById("root")
);

reportWebVitals();
```

<summary>Alternative Initial Set Up</summary>

You can preserve the `App.js` file in case you want to make abstractions, additional wrappers or configurations. It is not mandatory to delete it, but you have to make sure to delete the default `import App from './App'` generated by the `create-react-app` in order to avoid conflicts, or you can replace the `App` component in the `App.js` file with the one provided by neoAdmin like so:

```jsx title="/src/App.js"
import React from "react";
import { apps, contexts } from "@app-artisans/backoffice";

const { App: NeoAdminApp } = apps;
const { AuthProvider, ConfigProvider } = contexts;

function App() {
  return (
    <AuthProvider>
      <ConfigProvider>
        <NeoAdminApp />
      </ConfigProvider>
    </AuthProvider>
  );
}

export default App;
```

All the headers you will be creating in the project have to be imported to the `/src/pages/index.js`.

```jsx title="/src/pages/index.js"
import { headers as loginHeaders } from "./Login";

export const headers = [loginHeaders];
```

All the headers exported from `/src/pages/index.js` have to been passed to the `App` provided by the libary.

```jsx title="/src/index.js"
import React from "react";
import ReactDOM from "react-dom";
import reportWebVitals from "./reportWebVitals";
import { apps, contexts } from "@app-artisans/backoffice";
// highlight-next-line
import { headers } from "pages";

const { App } = apps;
const { AuthProvider, ConfigProvider } = contexts;

ReactDOM.render(
  <React.StrictMode>
    <AuthProvider>
      <ConfigProvider>
        // highlight-next-line
        <App headers={headers} />
      </ConfigProvider>
    </AuthProvider>
  </React.StrictMode>,
  document.getElementById("root")
);

reportWebVitals();
```

## Basic Login Page Example

:::info

By default all pages are set as authenticated. If you are interested in creating an unauthenticated page, you can use the <WrapperLink url="#">`route` prop `unAuth: true`</WrapperLink ><!-- /api/headers/options/route --> and set this prop in all the `headers.js` files.

```jsx
{
  type: "Page",
  options: {
    name: "Unathenticated Page",
    route: {
      path: "/",
      // highlight-next-line
      unAuth: true,
      auth: false,
      component: () => {},
    },
  },
};
```

:::

You can organize your project whathever you like, but we suggest you to create a folder `pages` under `src` and inside `pages` create a folder named as the corresponding page you want to create, in this case will be `Login`. Inside every page folder you have to create a `headers.js` and a `index.js` file. Like so:

You have to import the `Login` component in order to render the default Login page, but if you want you can create your custom page you only have to pass your custom page component to the `component` prop.

```jsx title="/src/pages/Login/headers.js"
import React from "react";
// highlight-next-line
import { pages } from "@app-artisans/backoffice";
import { loginRequest } from "utils/requests";

// highlight-next-line
const { Login } = pages;

const headers = {
  type: "Page",
  options: {
    name: "Login",
    route: {
      path: "/",
      unAuth: true,
      auth: false,
      exact: true,
      // highlight-next-line
      component: (props) => <Login {...props} onSubmit={loginRequest} />,
    },
  },
};

export default headers;
```

The `/src/pages/Login/index.js` file is a simple export file in order to make easier import all the headers in `/src/pages/index.js`.

```jsx title="/src/pages/Login/index.js"
export { default as headers } from "./headers";
```

## Basic CRUD Page Example

To build your frist CRUD you have to create a file inside `pages` and then create a folder named as the corresponding page you want to create, in this case will be `Posts`. Inside every page folder you have to create a `headers.js` and a `index.js` file. Like so:

```bash

|  â””â”€â”€ğŸ“ Posts
|     â”œâ”€â”€ğŸ“ headers.js
|     â””â”€â”€ğŸ“ index.js
// highlight-end
â””â”€â”€âœ¨ ...
```

Let's say you call to your API `https://yourapiurl.com/api/Posts/` and the response return is something like this:

```js
[
  {
    id: 1,
    image: "https://picsum.photos/200/300",
    title: "What is TDD and how to use it",
    date: "2022-11-14",
    content:
      "<h2>What is TDD? (Test Driven Development)</h2><br/><p>When dealing with a problem, wanting to fix a bug, refine an algorithm or refactor part of the code, it is important to know what is the expected result and what is the current status. Either via TDD or not, these two concepts have to be clear before you start writing code. Otherwise, we will find ourselves lost, improvising and even generating code that will not serve the purpose in question.</p>",
    category: "TDD",
    visible: true,
  },
  ...
];
```

Now you want to show the properties `id`, `title`, etc, you have to do it by specifing the `property: {propertyName}` inside the `fields` array.

```jsx title="/src/pages/Posts/headers.js"
const findAllPosts = () =>
  fetch("https://yourapiurl.com/api/Posts/").then((response) =>
    response.json()
  );

const headers = {
  type: "CRUD", // Specify the header as a `CRUD` type
  options: {
    name: "Post", // Name the sidebar option and the page title
    route: { path: "/posts", home: true }, // Specify the page url and make it the home page after signing in
    requests: {
      findRequest: findAllPosts,
    },
    tableOptions: {
      isEditable: true, // Enable showing edit rows button
      isDeletable: true, // Enable showing delete rows button
    },
  },
  sections: [
    {
      fields: [
        {
          property: "id", // Specify the prop you want to show
          upsertOptions: {
            show: false, // Do not show it on the create and edit page
          },
        },
        {
          property: "title",
        },
        {
          label: "Date",
          property: "date",
          type: "date",
        },
        {
          property: "visible",
          type: "checkbox",
        },
        {
          label: "Image",
          property: "image",
          type: "image",
          tableOptions: {
            show: false, // Do not show it on the table page
          },
        },
        {
          property: "content",
          type: "html", // Render the fully functional TinyMCE html editor
        },
        {
          property: "category",
          type: "multiselect",
          relation: {
            isMulti: false,
            name: "category",
            nameProps: ["category"],
            options: [
              { value: "code", label: "Code" },
              { value: "management", label: "Management" },
            ],
          },
        },
      ],
    },
  ],
};

export default headers;
```
